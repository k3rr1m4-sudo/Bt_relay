# Contribution Guidelines

Thank you for considering contributing to our project! Here are some guidelines to help you get started.

## Getting Started
1. **Fork the Repository**: Start by forking the repository to your own GitHub account.
2. **Clone the Repository**: Clone your fork to your local machine using:
   ```bash
   git clone https://github.com/yourusername/Bt_relay.git
   ```
   Replace `yourusername` with your GitHub username.
3. **Create a Branch**: Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b your-feature-name
   ```

## Development Setup
1. **Install Dependencies**: Make sure you have the required dependencies for the project installed on your local machine.
2. **Run the Application**: Follow the project's README for instructions on how to run the application locally.

## Submitting Changes
1. **Commit Your Changes**: Commit your changes with descriptive messages. For example:
   ```bash
   git commit -m "Add feature X"
   ```
2. **Push Your Changes**: Push your changes to your fork:
   ```bash
   git push origin your-feature-name
   ```
3. **Create a Pull Request**: Go to the original repository on GitHub and create a pull request from your forked repository.

## Code Style
- Follow the established coding conventions for this repository.
- Ensure that all code is properly formatted and free of linting errors before submitting your changes.

## Reporting Issues
- If you find a bug or have a feature request, please open an issue on GitHub. Provide as much detail as possible, including steps to reproduce the issue or a description of the feature you would like to see.

Thank you for your contributions!
